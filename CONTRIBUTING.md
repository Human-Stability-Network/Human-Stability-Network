# Contributing to Human Stability Network (HSN)

Thank you for your interest in contributing to HSN. This document outlines how you can participate in building human stability systems.

## Ways to Contribute

1. **Code & Technical Development** - Build the AI Value Engine, platforms, and tools
2. **Documentation** - Improve clarity, translation, and accessibility
3. **Program Design** - Help refine frameworks and workflows
4. **Community Partnership** - Connect HSN with local organizations
5. **Subject Matter Expertise** - Housing, education, healthcare, social work
6. **Funding & Grants** - Help sustain the ecosystem

## Contribution Principles

- **Dignity First** - Every solution must preserve human dignity
- **Privacy by Design** - Data protection is non-negotiable
- **Accessibility** - All systems must be usable by everyone
- **Sustainability** - AI Value Engine must fund operations
- **Transparency** - Open governance and decision-making

## How to Submit Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes following our standards
4. Run validation checks (if applicable)
5. Submit a Merge Request to the `main` branch
6. Complete the MR template (automatically loaded)

## Merge Request Requirements

- [ ] Description of change and rationale
- [ ] Alignment with HSN mission and philosophy
- [ ] Privacy and ethics review completed
- [ ] Documentation updated
- [ ] No sensitive data or hardcoded credentials

## Code of Conduct

All contributors must adhere to our [Code of Conduct](./CODE_OF_CONDUCT.md).
